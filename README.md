<!-- === 1 етап === -->
У таблиці campaigns зазначені айді компанії, канал комунікації,
ціль компанії, початок створення та кінець роботи компанії,
сегмент клієнтів на який спрямована компанія та очікуваний приріст.

У таблиці customers зазначені айді користувача, дата реєстрування, країна, вік, стать,
рейтинг лояльності, канал через який клієнт вперше зареєструвався.

У таблиці events зазначені айді івент, timestamp чітко зафіксована дата події, айді користувача, айді сесії, тип івенту,
тип девайсу, traffic_source джерело через який потратив користувач, campaign_id
Маркетингова кампанія, пов’язана з подією (0 означає відсутність кампанії), page_category
Категорія сторінки, на якій відбулася подія (Головна, PLP, PDP, Кошик, Оформлення замовлення)

У таблиці products зазначені категорія, бренд, айді продукта, ціна, дата поява продукта в каталозі,
тип продукту (преміум чи ні)

У таблиці transactions зазначені айді транзакції, точна дата події, айді користувача, айді продукта,
кількість одиниць товару в транзакції, розмір знижки, отриманий дохід від транзакції (від'ємний якщо повернення),
айді кампанії, чи було здійсненно повернення коштів (1 - повернення було, 0 - повернення не було)

primary key всі поля айді
